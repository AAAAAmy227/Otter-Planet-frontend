# 领水獭（1-2 分钟）设计方案 v0.1

> 目标：在主入口后，给孩子一个“马上拥有自己的水獭伙伴”的仪式化体验。风格采用扁平插画、明亮、活泼、奇幻。

## 1. 体验定位

- **情绪目标**：安全感（可爱）+ 惊喜感（魔法）+ 成就感（拥有专属伙伴）。
- **时长目标**：1-2 分钟内完成，不出现复杂填写。
- **交互原则**：全程“魔法棒引导”，单屏单决策，按钮大、文案短、反馈快。

## 2. 页面流（MVP）

### Step A：水獭召唤（约 20-30s）
- 主视觉：中央漂浮小岛 + 星光涟漪 + 水獭剪影。
- 文案：
  - 标题：`你的水獭正在赶来！`
  - 副文案：`挥动魔法棒，唤醒新伙伴`
- 操作：点击/拖动“魔法棒”一次触发召唤动画（粒子 + 光圈）。

### Step B：选择初始配件（约 30-40s）
- 只给 **3 组轻选择**，每组 3 个选项：
  1. 头饰（星星帽 / 小耳机 / 月亮发夹）
  2. 背饰（小背包 / 披风 / 气泡环）
  3. 手持（魔法棒 / 小书 / 发光贝壳）
- 交互：点击卡片即预览，支持“随机一下”。

### Step C：起名字（约 20-30s）
- 默认随机名（可一键换名）：`泡泡獭`、`星星獭`、`小浪花`。
- 允许手动输入（最多 8 字），但不强制。
- CTA：`完成领养`。

### Step D：绑定成功（约 10s）
- 反馈文案：`太棒了！<名字> 已经加入你的冒险队。`
- 展示：角色卡 + 初始属性（好奇心、创造力、勇气）+ `进入故事屋`按钮。

## 3. 视觉规范（参考图同风格）

## 3.1 色彩（建议）
- 主背景蓝：`#58C8FF` / `#2EA6F7`
- 点缀橙：`#FF9A3D`
- 荧光青：`#83F2FF`
- 草地绿：`#9EDB7D`
- 文本深蓝：`#1C3D66`

## 3.2 插画语言
- 线条：圆角外轮廓，统一 2-4px 视觉线重。
- 形状：大圆角、少锐角；岛屿/云朵/植物偏“果冻感”。
- 光效：少量发光边（荧光植物、魔法棒轨迹），避免过多粒子造成噪声。
- 层次：前景植物（发光）- 中景角色 - 远景漂浮岛，保持阅读焦点在角色。

## 3.3 动效基线
- 浮动：背景岛屿上下缓动（2-3s 循环）。
- 呼吸：水獭待机轻微缩放（1.5s）。
- 点击反馈：120-180ms 的缩放 + 光圈扩散。

## 4. 组件清单（前端落地）

- `OtterClaimWizard`
  - `SummonStage`
  - `AccessoryStage`
  - `NameStage`
  - `ResultStage`
- `OtterPreviewCard`（角色实时预览）
- `MagicWandCTA`（主交互按钮）
- `AccessoryOptionCard`（配件选项）

> 状态模型建议：
> `step`, `selectedAccessories`, `otterName`, `isSummoned`, `seed`

## 5. 文案草案（儿童友好）

- `挥挥魔法棒，召唤你的第一只小水獭！`
- `选一选它今天的冒险装备吧～`
- `给你的新伙伴起个好听的名字！`
- `准备好啦，去故事屋开始创作！`

## 6. 验收标准（本阶段）

- 1 分钟内可以完成“跳过命名”的极速领养。
- 每一步只出现 1 个主要行动按钮（不让孩子困惑）。
- 全流程可键鼠完成，移动端可点击完成。
- 视觉风格与参考图一致：扁平插画、亮色、奇幻漂浮感。
